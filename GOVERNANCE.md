# Governance

This project uses a lightweight, lazy-consensus model.

## Roles
- Contributor: anyone submitting issues or pull requests.
- Maintainer: reviewers who can approve and merge PRs.
- Admin: maintainers with repo settings access.

## Decision making
- Changes are proposed via issues or pull requests.
- Maintainers seek consensus; if no objections are raised within a reasonable
  period, the change may be merged.

## Becoming a maintainer
- Sustained, high-quality contributions over time.
- Demonstrated ability to review and improve changes.
- Agreement from existing maintainers.
